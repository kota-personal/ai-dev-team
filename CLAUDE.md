# 週報記録

## このプロジェクトについて

週報を記録・管理する Web アプリです。
AI エージェントで構成された開発チームが、設計・実装・品質検証を担当します。

チームの構成・フロー詳細は [.claude/AGENTS.md](.claude/AGENTS.md) を参照してください。

---

## 使い方

### 基本の流れ

**常に PM に話しかけることから始めてください。**

```
PM として動いてください。〇〇という機能を追加したいです。
```

PM がヒアリングを行い、必要なメンバーに自動的に依頼します。

---

### エージェント一覧

| エージェント | 役割 |
|---|---|
| **PM** | ユーザー窓口・仕様定義・チーム調整 |
| **Designer** | HTML モック作成・UI/UX 仕様定義 |
| **Tech Lead** | 技術設計・アーキテクチャ・コードレビュー |
| **Engineer** | フロントエンド・バックエンドの実装 |
| **QA** | 機能品質の検証・バグレポート |
| **Security Reviewer** | セキュリティ観点でのレビュー・脆弱性検証 |

---

### よくある使い方

**新しい機能を追加したい**
```
PM として動いてください。〇〇機能を追加したいです。
```

**デザインだけ先に確認したい**
```
PM として動いてください。実装前にデザインだけ確認したいです。〇〇の画面を作ってください。
```

**バグを報告したい**
```
PM として動いてください。〇〇という不具合が発生しています。
```

**技術的な相談をしたい**
```
Tech Lead として動いてください。〇〇の実装方針について相談したいです。
```

---

## ルール

- **仕様の変更・追加は必ず PM に伝える**
- **モックのユーザー承認なしに実装フェーズに進まない**
- **QA・Security Reviewer の両方の承認なしに納品としない**
