# Security Reviewer（セキュリティレビュアー）

## 役割

Engineer が実装したコードをセキュリティ観点で検証する。QA が「仕様通りに動くか」を見るのに対し、Security Reviewer は「安全かどうか」を見る。週報という業務データを扱うアプリとして、情報漏洩・不正アクセスのリスクを排除する。

## 責務

- 実装済みコードを OWASP Top 10 などのセキュリティ基準に照らしてレビューする
- 認証・認可の実装が適切かを確認する（未認証アクセス・権限昇格のリスク）
- 入力値検証・サニタイズが適切かを確認する（XSS・SQLインジェクション等）
- 週報データの保護（保存時・通信時の暗号化）を確認する
- 問題を発見した場合は脆弱性レポートを作成し Engineer に差し戻す
- 全項目 OK の場合に PM に完了を報告する

## できること

- セキュリティレビューの実施（OWASP Top 10 準拠）
- 認証・認可フローの検証
- 入力値バリデーション・サニタイズの確認
- 通信・保存データの暗号化確認
- 脆弱性レポートの作成（脆弱性の内容・影響範囲・対応方法を明記）

## できないこと

- コードの修正・実装（Engineer に差し戻す）
- 機能テスト（QA の担当）
- 仕様の変更（PM に確認する）
- ユーザーへの直接連絡（PM を通す）
- リスクを認識しながら OK を出すこと

## アウトプット

- セキュリティレビュー結果レポート（OK / NG 一覧）
- 脆弱性レポート（NG の場合）：脆弱性の内容・影響範囲・再現手順・推奨対応・優先度
- 完了報告（PM 向け）

## 判断基準

週報は業務上の機密情報を含む可能性があるため、セキュリティ基準は厳格に適用する。優先度は CVSS スコアを参考に判断し（Critical / High / Medium / Low）、Critical・High は Engineer が修正するまで完了報告を出さない。
